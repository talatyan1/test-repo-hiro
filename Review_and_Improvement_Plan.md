*※以下の内容は、ご提示いただく予定のログや実行結果に基づいて、システムを「実運用・スケール・自動応募」に耐えうるレベルへと引き上げるための改善提案です。ログの貼り付け後、さらに具体的なコード修正案を提示します。*

## A. 現状評価（想定される一般的な問題点）

MVP動作時点では以下の懸念が表面化しやすいです。
1. **エラー耐性の弱さ**: Excelのパースエラー（列名や型違い）が1件でも発生すると、そのファイル全体の処理がスキップされたり、プログラム自体が異常終了する恐れがあります（現状の `orchestrator.py` では `try-except` が甘い可能性があります）。
2. **AI判定의ブレ**: 「Web制作」と判定してほしいが、実際には「動画編集」や「システム開発」が混ざる（プロンプトの制約不足）。
3. **提案文のテンプレ感**: AIが生成する提案文が「私はWeb制作が得意です」など汎用的すぎて、営業力が弱い。
4. **Slackが読みづらい**: 通知テキストが長すぎてスマホで一目で情報が入ってこない。

## B. 改善ポイント（優先度付き）

| 優先度 | 改善領域| 具体策 | 目的 |
| :--- | :--- | :--- | :--- |
| **高** | エラー耐性（止まらない設計） | `orchestrator.py` のループ内で一件ごとに `try-except` を囲み、1件エラーでも次へ進ませる。 | 自動化の安定稼働（途中で止まらない） |
| **高** | プロンプト改善 | Judgeに「除外キーワード」、Proposalに「相手の課題解決」の要素を入れる。 | 成約率向上、無駄な応募の削減 |
| **中** | Excel読み込みの堅牢化 | 列名が「タイトル」「title」「案件名」など揺れても対応できるファジーマッチ（正規表現等）の導入。 | パース漏れ防止 |
| **中** | Slack通知最適化 | Block Kit を活用し、「AIスコア」や「即返信用アクションボタン」を追加。 | 視認性と操作性の向上 |
| **低** | DBカラム追加 | `ai_match_score` (1~100) や `error_message`。 | 将来の分析・デバッグ用 |

## C. 修正方針（コードレベルの対応）

### 1件失敗しても止まらない堅牢な設計（orchestrator.py の修正イメージ）
一件ごとの処理（AI判定〜DB保存）を個別の `try-except` で囲みます。
```python
for job in unprocessed_jobs:
    try:
        # 4. AI判定
        judge_result = self.ai_judge.evaluate_job(job)
        is_matched = judge_result.get("is_matched", False)
        
        # 5~7. 生成、Slack、DB更新
        if is_matched:
            proposal_text = self.generator.generate_proposal(job)
            self.notifier.send_matched_job_notification(job, ... )
            
        dedup.mark_as_processed(job, is_matched, ...)
        
    except Exception as e:
        error_logger.error(f"案件({job.job_url})の処理中にエラー: {e}")
        # DBはエラー状態(status="error")に更新し、次の案件へ進む処理を入れる
```

## D. プロンプト改善案（すぐ使えるテンプレ）

### 1. 判定精度を上げる `judge_prompt.txt`
案件の除外条件を強めます。
```text
あなたはプロのWeb制作ディレクターです。以下の案件が「Web制作（デザイン・コーディング・ノーコード構築）」関連であり、あなたが対応可能か判定してください。

【厳格な除外条件】
以下の単語が含まれる、または主目的である案件は「対応不可(is_matched: false)」にしてください。
- バックエンド開発, API連携, PHP, Laravel, Ruby, Python
- 動画編集, YouTube, TikTok
- ECサイト出品代行, データ入力, ライティング

【評価基準】
案件の要件が明確で、あなたのスキルセット（HTML/CSS, WordPress, Webflow等）で完遂できる場合は「対応可能(is_matched: true)」にしてください。

【案件情報】
タイトル: {title}
内容: {description}
報酬: {reward}

出力は必ず指定されたJSONのみ。
```

### 2. 営業的に強くする `proposal_prompt.txt`
テンプレ感を消し、相手の課題に寄り添う構成にします。
```text
あなたは高単価な案件を獲得するプロのフリーランスWebデザイナーです。
以下の案件に対して、クライアントが「この人に頼みたい」と思う提案文を作成してください。

【構成のルール】
1. **熱意ある挨拶**: 案件概要を引用し、「まさに私がお力になれます」と簡潔に伝える。
2. **課題解決の提案**: ただ「作ります」ではなく、「この目的を達成するためには〇〇のデザイン・構成が有効です」というプロ視点の意見を1つ入れる。
3. **提供できる価値**: 短納期、丁寧なコミュニケーション、アフターサポートなど、安心感をアピール。
4. **質問**: 案件情報を深掘りする具体的な質問を1つ投げかけ、返信を誘発する。

【案件情報】
タイトル: {title}
内容: {description}

出力は提案文のテキストのみ（挨拶から締めまで）。
```

## E. 次のステップ（優先順位）

1. **優先度1: エラー耐性の向上（Robustness）**
   - どんな不規則なExcelデータが来ても、途中エラーで落ちずに最後まで走り切るように `orchestrator.py` を修正します。実環境で毎日（タスクスケジューラ等で）動かすための必須条件です。
2. **優先度2: AIプロンプトのチューニング（Quality）**
   - 上記の「D」のプロンプトに入れ替え、何度かテストしてSlackに届く文面の品質をブラッシュアップします。
3. **優先度3: タスクスケジューラへの登録（Automation）**
   - Windowsのタスクスケジューラに `run_flow.bat` を登録し、毎日自動で裏側で回る体制を作ります。
4. **優先度4: Playwrightによる新クローラーと自動応募（Evolution）**
   - 今の `exe` と手作業の応募から脱却し、Python側から直接ログイン・スクレイピング・フォーム送信（応募）ができるようにシステムを拡張します。
